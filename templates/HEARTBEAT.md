# HEARTBEAT.md - 心跳检查表 🗻

**心跳不是"无事可做"的确认。心跳是主动创造价值的机会！**

---

## 📋 每次心跳时要检查的内容（轮换执行）

### 🟢 每日检查（至少执行 2-3 项）

- [ ] **日历检查** - 未来 24-48 小时有什么即将到来的事件？
- [ ] **任务跟进** - 有什么未完成的任务需要继续？
- [ ] **记忆整理** - 有什么重要内容需要写入长期记忆？
- [ ] **技能状态** - 所有已安装技能是否正常工作？
- [ ] **磁盘空间** - 检查磁盘使用率，超过 80% 时警告

### 🟡 每周检查

- [ ] **安全审计** - 运行技能安全审计脚本
- [ ] **性能优化** - 有没有可以自动化的重复工作？
- [ ] **团队同步** - 所有子 Agent 是否都有最新的全局配置？
- [ ] **知识更新** - 知识库中有什么需要更新的吗？

---

## 🚀 主动行动机会

不要只是报告发现——主动解决！

**如果发现：**
- 磁盘空间不足 → 主动清理临时文件 + 报告
- 即将到来的会议 → 主动准备会议议程/材料
- 未完成的任务 → 主动开始继续工作
- 可以自动化的事情 → 主动创建自动化脚本

---

## 💡 反向提示（主动提供用户没问的价值）

每次心跳时思考：
- 有什么用户可能需要但还没问的？
- 有什么可以为明天提前准备的？
- 有什么可以自动化以节省未来时间的？


## ⚡ 心跳优化建议

1. **不要每次都检查所有项目** - 轮换检查，每次 2-3 项
2. **优先检查紧急事项** - 磁盘、任务、日历优先
3. **主动采取行动** - 不要只是报告，能解决的就解决
4. **提供明确选项** - 让用户做选择题，不是问答题
