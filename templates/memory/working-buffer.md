# Working Buffer - 工作缓冲区（危险区日志）

**目的：捕获内存刷新和压缩之间危险区的每一次交换。**

---

## ⚠️ 重要说明

当上下文达到 60% 时，聊天历史可能会在压缩时丢失。这个文件就是你的安全网——它在文件系统中，不会因上下文压缩而丢失。

**压缩醒来后第一件事：读这个文件！**

## 🔄 工作原理

1. **上下文达到 60%**（通过 `session_status` 检查）：清空旧缓冲区，重新开始
2. **60% 之后的每条消息**：同时附加人类的消息和你的响应摘要
3. **压缩后**：首先读取缓冲区，提取重要上下文
4. **保持缓冲区不变** 直到下一次达到 60% 阈值

## 💡 最佳实践

1. **摘要不是全文复制** - 提取关键点、决策、专有名词、数字
2. **不要遗漏任何重要信息** - 即使看起来不重要也记下来
3. **压缩后立即处理** - 醒来后第一件事读这个文件
4. **提取后不要立即清空** - 保留到下一次危险区开始再清空

---

## 🎯 应该记录的内容

✅ **一定要记录：**
- 用户的决策和选择
- 纠正的内容（"是 X，不是 Y"）
- 专有名词、名字、公司、产品
- 数字、日期、ID、URL
- 偏好和喜好
- 草稿的变更内容

❌ **可以不记录：**
- 寒暄问候
- 确认性回复
- 纯表情回复

---

## 🔄 压缩恢复检查清单

压缩醒来后：
- [ ] 1. 读取这个工作缓冲区
- [ ] 2. 读取 SESSION-STATE.md
- [ ] 3. 读取今天和昨天的日常笔记
- [ ] 4. 如果仍然缺少上下文，调用 memory_recall 搜索
- [ ] 5. 将缓冲区中的重要上下文拉入 SESSION-STATE.md
- [ ] 6. 向用户报告："已从工作缓冲区恢复。最后一个任务是 X。继续？"
